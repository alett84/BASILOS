 This is a linker error telling you that you have two (or more) functions and/or variables with the same name at global scope in different object files. The most likely source of this error is an improper definition of a global variable. There is an important distinction between a variable {\itshape declaration} and a variable {\itshape definition} in C programs, especially at global scope. When you create a variable, like\+: \begin{DoxyVerb}int some_variable = 0;
\end{DoxyVerb}
 This is a {\itshape definition}. The name is reserved, as well as space in the object file to store its value. Contrast\+: \begin{DoxyVerb}extern int some_variable;
\end{DoxyVerb}
 This is a {\itshape declaration}. The {\ttfamily extern} keyword instructs the compiler to reserve the name, but not any space in the object file -\/ that space will be allocated elsewhere.

When placed in a header file, a {\itshape definition} attempts to reserve both the variable name {\itshape and} its space in every object file, resulting in the multiple definition error. In contrast, a {\itshape declaration} in a header file just allows you to access that variable, and can be {\ttfamily \#include}d in as many files as you want.

What you want to do for global variables is put the {\itshape declaration} (i.\+e. using the {\ttfamily extern} keyword, without an initial value) in your header file, then place the matching definition (without {\ttfamily extern}, and as good programming practice, with an explicit initial value) in exactly one source file.
